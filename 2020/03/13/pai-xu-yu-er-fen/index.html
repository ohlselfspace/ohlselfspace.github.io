排序

快速排序

快速排序作为经典的排序算法，基本思想源于分治法，其平均时间复杂度可以达到$O(nlogn)$，最坏时间复杂度为$O(n^2)$。

算法步骤：

  • 首先对于数组q[0~n-1]，分别设置指针i,j。其中指针i指向数组下标-1的位置，指针j指向数组下标n的位置。
  • 选定一个数temp作为标准，当i向右移，j向左移的过程中，指针i停留在大于等于temp的位置，指针j停留在小于等于temp的位置，这样就可以保证指针i左边的元素都是小于temp的，指针j右边的元素都是大于temp的。然后当i<j时，交换q[i]和去q[j]。
  • 递归调用左右区间
void quick_sort(int *q , int l, int r){
    if(l >= r) return; //递归调用时边界条件

    int temp = q[l + r >> 1], i = l - 1, j = r + 1; //选取中间的数防止边界溢出和加深递归栈的深度
    while(i < j){
        do i++; while(q[i] < temp);
        do j--; while(q[j] > temp);
        if(i < j) swap(q[i], q[j]);
    }

    //递归调用
    quick_sort(q, l, j);
    quick_sort(q, j + 1, r);
}

tips:

注意上面递归调用时，存在两种调用情况容易发生错误：

  • /*
    这里temp不能取q[r]，会发生死循环
    样例：1 2
    当i到达下标1的时候，j也在下标1。此时进行递归调用
    quick_sort(q, l, j) ——>依旧调用的是[0,1]区间发生死循环
*/
quick_sort(q, l, j);
quick_sort(q, j + 1, r);

/*
    这里temp不能取q[l]，情况如下文相同
*/
quick_sort(q, l, i - 1);
quick_sort(q, i, r);

//为了避免以上情况发生，所以取temp为q[l + r >> 1]

归并排序

归并排序的思想基于分治法，时间复杂度$O(nlogn)$。

算法步骤：

  • 首先确定分界点mid=（r+l）/2，把区间划分为左右两个区间。
  • 对左右两个区间分别递归进行划分，划分的左右两个区间的元素一定有序。
  • 合并，采用双指针算法对左右区间进行扫描，比较两个指针指向的元素，把较小的元素加入辅助数组。直到其中一个区间被扫描完毕，然后把另一个区间加到辅助数组的末尾即可。
int temp[r + 1]; //辅助数组
void merge_sort(int *q, int l , int r){
    if(l >= r) return;

    //划分
    int mid = r + l >> 1;
    merge_sort(q, l , mid);
    merge_sort(q, mid + 1, r);

    //合并
    int k=0, i = l, j = mid + 1;
    while(i <= mid && j <= r){
        if(q[i] < q[j]) temp[k ++] = q[i ++];
        else temp[k ++] = q[j ++];
    }
    while(i <= mid ) temp[k ++] = q[i ++];
    while(j <= r) temp[k ++] = q[j ++];

    //把辅助的值赋值给原数组q
    for(i = l, j = 0; i <= r; i ++ , j ++) q[i] = temp[j];
}

二分法

整数二分

二分的本质是根据具体的问题将区间[l,r]分为两半，其中左边部分不满足某种性质，而右边部分满足某种性质，所以通过二分能够找到两个区间的分界点。通常情况下，整数二分将区间划分为左边部分是[l,mid]，右边部分是[mid+1,r]进行递归处理。

算法模板1（找到左边部分的边界）：

  • 首先让mid=（l+r+1）/2，检查mid是否满足左边部分的性质。
  • 如果为真，那么说明左边部分的边界一定在mid右边（包括mid），此时更新l=mid。反之，如果为假的话，说明mid不满足左边部分的性质，此时左边部分的边界一定在mid左边（不包括mid），此时更新r=mid-1。
  • 重复更新mid，直到l>=r跳出循环，此时左边部分的边界在l。
/*
    在寻找左边部分的边界时，需要注意：如果mid=(l+r)/2,当l=r-1时，r与l相邻，
    此时如果满足性质的话，mid=l,l=mid -->此时就会陷入死循环
    所以应该上取整，取mid=(r+l+1)/2
*/
int binary1(int l , int r){
    while(l < r){
        int mid = l + r + 1 >> 1;
        if(check(mid)) l = mid;
        else r = mid - 1;
    }
    return l;
}

算法模板2（找到右边部分的边界）：

  • 首先让mid=（l+r）/2，检查mid是否满足右边部分的性质。
  • 如果为真，那么说明右边部分的边界一定在mid左边（包括mid），此时更新r=mid。反之，如果为假的话，说明mid不满足右边部分的性质，此时右边部分的边界一定在mid右边（不包括mid），此时更新l=mid+1。
  • 重复更新mid，知道l>=r跳出循环，此时右边部分的边界在l。
int binary2(int l, int r){
    while(l < r){
        int mid = l + r >> 1;
        if(check(mid)) r = mid;
        else l = mid + 1;
    }
    return l;
}

tips:

整数二分的关键在于区间边界的处理。

案例 数的范围

给定一个按照升序排序的长度为n的整数数组，进行查询，返回查询元素的起始位置和终止位置。

//核心逻辑

//起始位置-->寻找右边部分的边界(右边边界的性质是大于等于x)
while(l < r){
    int mid = l + r >> 1;
    if(q[mid] >= x) r = mid;
    else l = mid + 1;
}

//x不存在时，二分的位置为从左往右第一个大于等于x的数
if(q[l] != x) printf("no exist!");
else{
    printf("起始边界：%d",l);
    //终止位置-->寻找左边部分的边界(左边边界的性质是小于等于x)
    int l = 0, r = n - 1;
    while(l < r){
        int mid = l + r + 1 >> 1;
        if(q[mid] <= x) l = mid;
        else r = mid - 1;
    }
    printf("终止边界：%d",l);
}

小数二分

小数二分问题比较简单，没有处理边界的问题。

算法模板：

double binary3(int l, int r){
    double eps = 1e-6;
    while(r - l > eps){
        double mid = (l + r) / 2;
        if(check(mid)) r = mid;
        else l = mid;
    }
    return l;
}
